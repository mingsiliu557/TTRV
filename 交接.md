64 rollout 实验回应了“是否是 rollout 太少导致伪标签由少量 candidate 决定”的问题：不是主要原因。把 rollout 加到 64 后，很多错误多数仍然高度稳定，说明问题更像是模型先验和选项语义/标注口径错位。典型 bad case：

- `physxnet_mcq_00000005`: 选项同时包含 `A. revolute joint` 和 `D. hinge joint`，标注为 A，但模型稳定选 D。hinge 常被理解为 revolute 的一种，Geo-Harmony 会把这种稳定错误当高置信伪标签。
- `physxnet_mcq_00021341`: 标注为 hinge，但模型稳定选 revolute；也是 hinge/revolute 口径冲突。
- `physxnet_mcq_00042676`: drawer 类样本中，标注 prismatic，但模型大量输出 hinge/revolute，说明 PointLLM 对形状/部件类别有强先验，单靠自一致 reward 会放大这个先验。

为验证这个判断，代码里临时加了一个 reward 侧门控：

```text
GEO_SKIP_AMBIGUOUS_JOINT_OPTIONS=True
```

该门控只看 prompt 选项文本，不看 ground truth；当 `joint_type` 选项同时出现 `hinge/revolute`、`slider/prismatic`、`fixed/rigid` 等近义冲突时跳过伪标签 reward。它能避免部分明显坏信号，但会导致训练信号非常稀疏，而且仍无法解决非跳过样本更新后全量 acc@1 下滑的问题。因此当前不建议继续扩大这条无标签权重更新路线。

### Reward / parsing

主路径保持：

```text
prediction_raw -> parse_prediction(prediction_raw) -> A/B/C/D/unknown
```

规则：

- 训练 reward 和频率统计都基于 parsed prediction，不使用 raw response 字符串。
- `unknown` 作为 invalid/incorrect 处理。
- 纯 TTRV/Geo-Harmony reward 不使用 ground truth；ground truth 只用于 eval 和分析。
- `UNKNOWN_REWARD=0.0` 对 fragile 类型更稳定；负 unknown reward 在 joint/movable 早期实验中更不稳。

Geo-Harmony 当前做法：

- 同一 prompt 采样多个 response。
- 先解析成 A/B/C/D/unknown。
- 基于 parsed prediction 计算频率、归一化熵、view consistency。
- 使用 `TTRL_MIN_MAJORITY_RATIO` / `TTRL_MAX_MAJORITY_RATIO` 过滤伪标签。
- 训练日志记录 reward、majority ratio、entropy、invalid、hitmax 等 collapse 信号。

| policy | 当前实现 |
| --- | --- |

| `sparsity` | 随机 downsample 后重采样回原点数 |
| `rigid_pose` | scale + translation |

注意：

- 当前 `rigid_pose` 没有 rotation，只实现 scale + translation。
- `sensor_noise` 不使用 translation/downsample。
- `sparsity`, `rigid_pose`, `composite`, `mixed`, `renormalize=True` 都还没有形成系统 full-data 消融结论。

## 参考文献和方法关系

### TTRL

```text
Yuxin Zuo et al. TTRL: Test-Time Reinforcement Learning.
arXiv:2504.16084
https://arxiv.org/pdf/2504.16084
```

关系：

- TTRL 的核心设定是无标签 test-time RL，用模型自身在 test-time scaling 中产生的信号做 reward。
- 论文中使用 majority-vote / maj@n 这类自监督信号作为训练奖励来源。
- 我们最早的 PhysX-MCQ majority reward 实验基本对应这个方向：对同一 prompt 多次采样，解析到 A/B/C/D/unknown，再用 majority pseudo-label 奖励。
- 在 PhysX-MCQ 上观察到的问题是：PointLLM 的稳定输出不一定正确，majority reward 会强化稳定但错误的选项，因此全量 `valall` 上出现 step 后 acc@1 下降。

### Self-Harmony

```text
Ru Wang et al. Self-Harmony: Learning to Harmonize Self-Supervision and Self-Play in Test-Time Reinforcement Learning.
arXiv:2511.01191
https://arxiv.org/pdf/2511.01191
```

关系：

- Self-Harmony 针对 TTRL 中 majority voting 容易偏向 spurious popular answers 的问题，引入 original / reframed views 的一致性。
- 它用同一个模型同时扮演 Solver 和 Reframer，通过原问题和重写问题的答案频率 harmonic mean 来选择更稳定的 pseudo-label。
- 我们当前的 Geo-Harmony 是面向 3D MCQ 的类比改造：没有使用文本 paraphrase，而是用 point-cloud reframe views 产生多视角扰动，再在 parsed MCQ prediction 上计算频率/熵/一致性。
- 当前最终保留实验只系统使用了 `sensor_noise` view；其他 point-cloud transform 还需要继续消融，尤其要用 acc@1 step validation 判断是否真的改善训练，而不是只看 vote8 majority。

### RESTRAIN / soft-label reward 方案

用户建议参考：

```text
RESTRAIN: From Spurious Votes to Signals -- Self-Driven RL with Self-Penalization
arXiv:2510.02172
https://arxiv.org/abs/2510.02172
https://arxiv.org/pdf/2510.02172
```

论文要点和我们问题高度相关：

- RESTRAIN 明确指出 majority vote 可能是 spurious，正确答案可能在 minority rollouts 中。
- 它不是把所有质量压到 majority answer，而是利用整个 answer distribution。
- 核心包括：
  - pseudo-label weighting：按 vote count 形成 pseudo-label distribution；
  - negative rollout penalization：低一致性 prompt 不直接信 majority，使用零 reward/负 advantage offset；
  - prompt-level weighting：用 frozen/reference model 的一致性给 prompt 降权，避免训练中自信度反馈循环。
- 论文的 ablation 显示：去掉 pseudo-label weighting 会不稳定甚至 collapse；均匀 soft label 也会更差，因为低频噪声被过度放大。

对我们当前现象的判断：

```text
soft-label reward 值得作为下一轮主线测试。
```

原因：

- 当前 hard label 把 `C6/D1/unknown1` 全部 reward 压到 C，导致 answer D 的少数正确样本被负相对优势压下去。
- soft vote distribution 至少可以避免“一票定生死”的硬坍塌，把梯度从 one-hot majority 改为按 vote count 加权。
- 但它不能保证解决 `A6/B1/C1` 这种多数稳定错的问题；如果错误多数很强，soft distribution 仍然会主要强化错答案，只是力度更缓。
- 因此不能只做 soft label，还要保留 prompt gate / low-consistency skip / early-stop recovery。

### 下一轮建议实现：RESTRAIN-lite for PhysX-MCQ

不要直接覆盖当前 hard-label 方法。新增 reward variant，先小步测试：

```text
REWARD_VARIANT=geo_harmony_soft
```

候选设计：

1. 对每个 prompt 的 8 个 parsed predictions 得到 counts：

```text
q(a) = count(a) / known_count, a in A/B/C/D
unknown 不进入 label distribution；unknown rollout reward=UNKNOWN_REWARD 或 0
```

2. 可选 skew/temperature，避免均匀化低频噪声：

```text
q_sigma(a) = softmax(log(count(a)+eps) / sigma)
建议先 sigma=0.5
```

或更简单：

```text
q_power(a) = count(a)^gamma / sum_b count(b)^gamma
建议 gamma=1.5 或 2.0
```

3. 每条 rollout 的 reward 不再是 hard pseudo-label 的 0/1，而是该 rollout 预测对应的 soft mass：

```text
reward_i = q(pred_i)
```

4. 结合 Geo-Harmony 时，不用单个 `geo_label`，而是把各 view 的 label distribution 做平均/几何平均/调和加权：

```text
per_view_prob_v(a) = count_v(a) / samples_per_view
geo_soft_score(a) = harmonic_mean_v(per_view_prob_v(a) + eps)
q_geo(a) = normalize(geo_soft_score(a)^gamma)
```

5. prompt-level gate：

```text
if max_count < kappa:
    all rewards = 0
```

先不要做负 advantage offset；veRL/GRPO 里直接做负 advantage offset 需要更小心改 trainer。第一版先用 zero-reward skip + early stop。
